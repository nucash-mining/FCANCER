Staking Process

// 1. Approve token spending await stakingToken.approve(stakingContract, amount); // 2. Stake tokens await stakingContract.stakeTokens(amount); // 3. Mine blocks to generate rewards await stakingContract.mineBlock(); // 4. Claim accumulated rewards await stakingContract.claimRewards();
Important: Rewards are calculated based on the global staking pool across all networks. Cross-chain synchronization may take a few minutes to update.

Cross-Chain Synchronization

The system uses cross-chain messaging to synchronize staking amounts across all networks:

  1. User stakes tokens on any supported network
  2. Contract broadcasts stake update to all other networks
  3. Global staking pool is updated across all chains
  4. Rewards are calculated proportionally

üåâ wALT Bridge System

Bridge Overview

The wALT bridge enables seamless transfer of ALT tokens between Altcoinchain and wrapped ALT (wALT) tokens on other networks.

üîí Security Features

  • Rate Limiting: 100,000 wALT daily limit per address
  • Cooldown Period: 5-minute cooldown between bridges
  • Bridge Fee: 0.05% fee for network security
  • Authorized Relayers: Multi-signature relayer network
  • Emergency Pause: Circuit breaker for security incidents

Bridge Process

// 1. Approve wALT spending await waltToken.approve(bridgeContract, amount); // 2. Initiate bridge with message fee const messageFee = await bridgeContract.getMessageFee(); await bridgeContract.initiateBridge(amount, targetChainId, { value: messageFee }); // 3. Wait for cross-chain message processing // 4. Tokens appear on target chain (2-10 minutes)

üöÄ Deployment Guide

Prerequisites

  • Node.js 16+
  • Hardhat development environment
  • Private key with sufficient gas on target networks
  • RPC endpoints for all target networks

Environment Setup

# Clone repository git clone https://github.com/nucash-mining/FCANCER/ cd FCANCER # Install dependencies npm install # Create environment file cp .env.example .env # Add your private key and RPC URLs

Deployment Commands

# Deploy to single network npx hardhat run scripts/deployCrossChainStaking.js --network altcoinchain # Deploy to all networks npm run deploy:crosschain-staking:all # Deploy wALT bridge with deterministic addresses npm run deploy:walt-bridge:deterministic # Deploy all wALT bridges npm run deploy:walt-bridge:all

Verification

After deployment, verify contracts on block explorers:

# Verify on Etherscan (example) npx hardhat verify --network ethereum CONTRACT_ADDRESS "CONSTRUCTOR_ARG1" "CONSTRUCTOR_ARG2"
Security Note: Always verify contract source code on block explorers before interacting with deployed contracts.

üîß Troubleshooting

Common Issues

Staking Issues

  • Insufficient Allowance: Approve token spending before staking
  • Network Mismatch: Ensure you're connected to the correct network
  • Pending Rewards: Mine blocks to generate new rewards

Bridge Issues

  • Rate Limiting: Wait for cooldown period or check daily limits
  • Insufficient Message Fee: Ensure sufficient ETH for cross-chain messages
  • Network Congestion: Bridge may take longer during high traffic

General Issues

  • Transaction Failures: Check gas limits and network status
  • Contract Not Found: Verify contract addresses for your network
  • RPC Errors: Try switching to a different RPC endpoint
Support: For additional help, join our community Discord or check the GitHub repository for the latest updates and known issues.
